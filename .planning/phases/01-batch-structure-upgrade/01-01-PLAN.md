<objective>
把現存「批次結構」程式碼的最後幾哩補齊：
1. 統一 defects 讀取路徑（D3）：所有 render/統計/匯出邏輯一律從 `summary.defects` 讀；保留 submitStation 的雙寫（頂層 defects + summary.defects）。
2. 補強 product 欄位的 runtime fallback（D2）：所有讀取 `r.product` 的位置都要能容忍舊資料（fallback `'重乳'`）。
3. 在管理頁工單匯入／預覽 UI 顯示 product 下拉（D4）：選項僅「重乳」一個，為 V3 多品項預留位置。
4. 確認批次卡片（看板／工站選層）顯示 product 標籤。

Purpose: 讓前端程式碼與 v2.1 schema 一致，並建立未來移除頂層 defects 與引入多品項時不需動到 render layer 的基礎。
Output: 修改後的 `app.js` 與必要的 `index.html` 結構調整。

不在範圍內：
- 多層複選 UI（Phase 2）
- Sheet URL 拉取流程（Phase 2，現況已實作完成）
- 移除頂層 `defects` 寫入（per D3 維持雙寫）
- 「清除本地資料」按鈕（per CONTEXT Deferred ideas，排 Phase 2）
</objective>
